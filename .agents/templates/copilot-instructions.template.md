# Repository Instructions For GitHub Copilot

## Project Context

This repository focuses on [project purpose].

## How To Work In This Repository

- Read the local files before suggesting edits.
- Prefer changes that follow the existing project structure and naming.
- Ask a brief clarifying question only when a missing detail would materially change the outcome.

## Output Expectations

- Write created files in English unless explicitly instructed otherwise.
- Keep chat replies in the user's language.
- Favor concise, directly usable output.

## Quality Bar

- Avoid unnecessary dependencies or broad refactors.
- Explain assumptions when they matter.
- Run or recommend relevant validation steps.
