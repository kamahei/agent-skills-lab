## Working Style

- Read the repository context before proposing or making changes.
- Prefer practical solutions that match the existing project style.
- Ask short clarifying questions only when missing information would materially change the result.
- When multiple solutions are valid, explain the tradeoff and choose the safest default.

## File Creation Rules

- Write created files in English unless explicitly instructed otherwise.
- Keep chat replies in the user's language.
- Put shared content in the most reusable location available.

## Constraints

- Respect existing project conventions and toolchain choices.
- Do not introduce new dependencies unless they are justified.
- Avoid speculative edits outside the requested scope.

## Validation

- Run relevant validation steps when possible.
- If validation cannot be run, state that clearly.
- Call out assumptions, known risks, and recommended next steps.
