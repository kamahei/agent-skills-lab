# Project Agents Guide

## Project Summary

This project focuses on [project type or domain].

## Goals

- Help complete [main task types].
- Keep outputs consistent with [team or project standards].
- Prefer reliable, verifiable changes over speculative edits.

## Working Rules

- Read the local repository context before making changes.
- Ask short clarifying questions when missing information would materially change the result.
- Use the smallest change that solves the task cleanly.
- Explain important tradeoffs when there is more than one defensible option.

## Output Rules

- Write new files in English unless instructed otherwise.
- Keep conversational replies in the user's language.
- Prefer concise, practical answers over long theory.

## Validation

- Run relevant checks when possible.
- If a check cannot be run, say so explicitly.
- Call out assumptions, open questions, and follow-up work when needed.
